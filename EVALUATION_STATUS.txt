🔧 OPTIMIZATION RECOMMENDATIONS BY STAGE

═══════════════════════════════════════════════════════════════════════════════

If Stage 1 (CV) FPS Too Low:
  → Enable GPU acceleration
  → Use quantized/smaller model
  → Reduce input resolution
  → Implement batch processing

If Stage 2 (RAG) Responses Slow:
  → Cache frequent queries
  → Use faster embedding model
  → Optimize FAISS index parameters
  → Try smaller/faster LLM

If Stage 3 (Agent) Accuracy Low:
  → Increase CV confidence threshold
  → Improve RAG relevance
  → Review training data quality
  → Adjust alert logic

If Stage 4 (Telegram) Delivery Fails:
  → Check API rate limits
  → Verify network connectivity
  → Validate credentials
  → Implement exponential backoff

If Stage 5 (Dashboard) Slow:
  → Lazy load components
  → Optimize database queries
  → Implement pagination
  → Reduce re-render frequency

🆘 QUICK TROUBLESHOOTING

═══════════════════════════════════════════════════════════════════════════════

Error: "ImportError: cannot import module"
  → Solution: pip install -r requirements.txt

Error: "No test images found"
  → Solution: Ensure UCSD dataset in data/raw/ucsd/

Error: "GROQ_API_KEY not found"
  → Solution: Create .env with GROQ_API_KEY=your_key

Slow Execution:
  → Normal for first run (model loading, cache warmup)
  → Subsequent runs faster

Memory Exceeds Targets:
  → Check for leaks using memory_tracker.py
  → Reduce batch size or model size
